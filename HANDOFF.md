# HANDOFF

## Purpose
This file records short-lived implementation context when work stops in the middle of a problem and fast resumption matters.

Use it for:
- partially implemented features
- debugging dead ends
- temporary workarounds
- commands that failed
- file-specific cautions for the next session

```md
## Topic
<feature or bug>

## Current state
<what is already done>

## What was tried
<attempts, commands, observations>

## Blocker or risk
<why work stopped>

## Affected files
- <path>

## Next recommended step
<best next action>
```
